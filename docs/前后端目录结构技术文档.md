# 前后端目录结构技术文档

## 1. 文档用途

这份文档只说明三件事：

1. 项目目录怎么分
2. 关键文件各自负责什么
3. 开发和打包入口在哪里

---

## 2. 项目结构

```text
personal_blog/
├─ Makefile                     # 统一构建、检查、打包、部署入口
├─ frontend/                    # React 前端工程
│  ├─ public/                   # 静态资源
│  ├─ src/
│  │  ├─ components/            # 通用组件
│  │  ├─ hooks/                 # 页面与业务 hook
│  │  ├─ pages/                 # 页面模块
│  │  ├─ router/                # 前端路由
│  │  ├─ services/              # 前端请求封装
│  │  ├─ App.tsx                # 前端应用壳
│  │  ├─ main.tsx               # 前端启动入口
│  │  └─ styles.css             # 全局样式
│  ├─ .env.example              # 前端环境变量模板
│  ├─ index.html                # Vite HTML 入口
│  ├─ package.json              # 前端依赖与脚本
│  ├─ tsconfig.json             # TypeScript 配置
│  └─ vite.config.ts            # Vite 配置
├─ backend/                     # FastAPI 后端工程
│  ├─ app/
│  │  ├─ api/
│  │  │  ├─ endpoints/          # 接口模块
│  │  │  └─ router.py           # 接口聚合入口
│  │  ├─ core/
│  │  │  ├─ runtime.py          # 打包运行时路径解析
│  │  │  └─ settings.py         # 配置读取
│  │  ├─ schemas/               # 请求/响应模型
│  │  ├─ services/              # 后端业务服务
│  │  ├─ main.py                # FastAPI 应用入口
│  │  └─ web.py                 # 前端静态资源托管
│  ├─ build/                    # PyInstaller 中间产物
│  ├─ dist/                     # 打包输出目录
│  ├─ .env.example              # 后端环境变量模板
│  ├─ personal_blog.spec        # PyInstaller 配置
│  ├─ pyproject.toml            # 依赖与工具配置
│  ├─ run.py                    # 程序运行入口
│  └─ uv.lock                   # 依赖锁文件
├─ docs/
│  ├─ 前后端目录结构技术文档.md
│  └─ 一键部署平台文档.md
└─ README.md
```

## 3. 关键文件职责

### 3.1 前端

| 路径 | 作用 |
|---|---|
| `frontend/src/main.tsx` | 挂载前端应用 |
| `frontend/src/App.tsx` | 应用整体壳层 |
| `frontend/src/router/` | 管理前端页面路由 |
| `frontend/src/pages/` | 按页面拆分业务界面 |
| `frontend/src/services/` | 管理接口请求 |
| `frontend/src/hooks/` | 收纳可复用状态逻辑 |

### 3.2 后端

| 路径 | 作用 |
|---|---|
| `backend/app/main.py` | 创建 FastAPI 应用 |
| `backend/app/api/router.py` | 聚合所有接口路由 |
| `backend/app/api/endpoints/` | 按业务拆接口 |
| `backend/app/core/settings.py` | 读取环境变量与默认配置 |
| `backend/app/core/runtime.py` | 兼容源码运行和 PyInstaller 运行 |
| `backend/app/web.py` | 挂载前端静态文件和 SPA fallback |
| `backend/run.py` | 程序主入口 |
| `backend/personal_blog.spec` | 控制打包内容 |

### 3.3 根目录

| 路径 | 作用 |
|---|---|
| `Makefile` | 统一打包、检查、部署命令 |
| `README.md` | 项目总说明 |
| `docs/一键部署平台文档.md` | 部署与运维说明 |

## 4. 开发入口

### 4.1 前端开发

```bash
cd frontend
npm install
npm run dev
```

### 4.2 后端开发

```bash
cd backend
uv sync
uv run uvicorn app.main:app --reload --host 0.0.0.0 --port 7869
```

---

## 5. 打包入口

Ubuntu 服务器源码目录执行：

```bash
make package
```

打包完成后产物位置：

```text
backend/dist/personal_blog
```

回归测试：

```bash
make smoke
```

---

## 6. 维护规则

1. 前端页面代码优先放在 `pages/`，通用逻辑放 `hooks/`。
2. 后端接口代码优先放在 `app/api/endpoints/`，不要把所有接口堆进一个文件。
3. 打包逻辑统一收口到 `Makefile`、`personal_blog.spec`、`app/core/runtime.py`。
4. 配置优先走 `.env`，不要到处硬编码路径和端口。
5. 单个模块尽量控制在 600 行以内。

---

## 7. 什么时候拆模块

满足任一条件时，应该继续拆分：

1. 单文件超过 500 到 600 行
2. 同一类逻辑在多个文件重复出现
3. 前后端某个目录已经开始难找、难改、难维护
