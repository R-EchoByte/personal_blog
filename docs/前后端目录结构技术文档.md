# 前后端目录结构技术文档（简洁清晰版）

## 1. 文档目标

- 前端统一使用 React（Vite + TypeScript）。
- 后端统一使用 FastAPI。
- 目录结构遵循“少层级、单入口、低耦合”。
- 直接对接 1Panel 的部署与运维流程。

## 2. 快速导航

1. 先看“第 3 章目录总览”，确认文件位置。
2. 再看“第 4 章职责矩阵”，理解每个关键文件做什么。
3. 按“第 5 章本地启动顺序”运行项目。
4. 上线前阅读“第 6 章 1Panel 对接”。

## 4. 关键文件职责矩阵

| 路径 | 角色 | 责任边界 | 何时修改 |
|---|---|---|---|
| `frontend/src/main.tsx` | 前端启动入口 | 挂载根节点与路由提供器 | 更换全局 Provider 时 |
| `frontend/src/router/index.tsx` | 前端路由总入口 | 集中定义页面路由，不做业务逻辑 | 新增页面路由时 |
| `frontend/src/services/api.ts` | 前端请求入口 | 统一请求基地址与错误处理 | 增加鉴权头、重试策略时 |
| `backend/app/main.py` | 后端启动入口 | 创建 FastAPI 实例并挂载 API 路由 | 接入中间件或生命周期事件时 |
| `backend/app/api/router.py` | 后端路由聚合 | 统一注册各业务 endpoint | 新增后端业务模块时 |
| `backend/app/core/settings.py` | 配置管理 | 统一读取 `.env` 与默认值 | 新增配置项时 |

## 5. 本地启动顺序

## 5.1 启动前端

1. `cd frontend`
2. `npm install`
3. `npm run dev`

## 5.2 启动后端

1. `cd backend`
2. `python -m venv .venv`
3. `.venv\\Scripts\\activate`
4. `pip install -r requirements.txt`
5. `uvicorn app.main:app --reload --port 8000`

## 5.3 联调检查

1. 打开 `http://localhost:5173`
2. 打开 `http://localhost:8000/`
3. 打开 `http://localhost:8000/api/v1/health`

## 6. 1Panel 对接（最小步骤）

1. 前端执行 `npm run build`，发布 `frontend/dist` 到站点静态目录。
2. 后端以容器运行 FastAPI，建议端口 `8000`。
3. Nginx 反向代理 `/api` 到后端容器。
4. 前端环境变量 `VITE_API_BASE_URL` 指向线上 API 前缀。
5. 后端通过 `.env` 注入数据库、Redis、密钥等配置。

## 7. 变更规则（保持简洁）

1. 新增页面只改 `pages` 与 `router`，不要增加新顶层目录。
2. 新增后端业务只在 `app/api/endpoints` 增文件并在 `api/router.py` 注册。
3. 复杂度未达到前，不引入 `repository/service` 额外分层。
4. 一切配置项优先进入 `.env`，避免硬编码。

## 8. 扩展触发条件

当满足任一条件时，再引入更细分层：

1. 单个业务模块超过 500 行且多人协作频繁冲突。
2. 相同数据访问逻辑在 3 个以上 endpoint 重复出现。
3. 接口数量超过 30 且需要分团队维护。
