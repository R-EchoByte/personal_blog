# 首页首屏实现说明（React）

## 1. 目标

- 首屏布局参考示例图：全屏背景、中心视觉文案、毛玻璃语录卡片、底部快捷入口与版权信息。
- 保持结构简单，前端仅在一个页面文件和一个样式文件内完成核心实现。

## 2. 实现文件

- 页面组件：`frontend/src/pages/HomePage.tsx`
- 全局样式：`frontend/src/styles.css`
- 背景图目录：`frontend/public/`（文件名需以 `bj` 开头）
- 背景清单：`frontend/public/bj-manifest.json`（由脚本自动生成）
- 清单脚本：`frontend/scripts/generate-bg-manifest.mjs`

## 3. 布局结构

```text
landing-page
├─ landing-toolbar（个性标签 + 背景控制）
├─ landing-backdrop（额外光晕层）
├─ landing-content
│  ├─ brand-badge（Logo 占位）
│  ├─ landing-title（主标题）
│  ├─ landing-subtitle（副标题）
│  ├─ quote-card（语录卡片）
│  └─ quick-links（快捷入口卡片）
└─ landing-footer（版权信息）
```

## 4. 样式要点

- 背景：图片层 + 渐变遮罩层叠加，保证文字可读性。
- 顶部工具栏：展示个人技术标签、日期信息、背景状态与操作按钮。
- 卡片：`background: rgba(...) + backdrop-filter: blur(...)` 实现毛玻璃。
- 文字：主标题、语录、作者分级展示，保留清晰视觉层次。
- 响应式：在 `@media (max-width: 900px)` 下缩放徽标、卡片内边距、字号与底部版权条。

## 5. 可配置项

- 快捷入口：修改 `HomePage.tsx` 中 `quickLinks` 数组。
- 语录文本：修改 `quote-text` 与 `quote-author` 内容。
- 背景图：
  - 把图片放到 `frontend/public/`
  - 文件命名为 `bj*.jpg|jpeg|png|webp|avif`，例如 `bj1.jpg`、`bj_sky.webp`
  - 页面刷新会从这些 `bj*` 图片中随机选择一张
- 背景切换交互：
  - 点击页面右上角按钮 `换背景 (B)`
  - 键盘按 `B` 触发即时切换
- 语录切换交互：
  - 页面加载时自动请求随机语录
  - 点击语录卡片右上角按钮 `换一句 (Q)`
  - 键盘按 `Q` 触发即时切换
  - 后端限制语录最长 16 个字
- 模块直达快捷键：
  - `1` 跳转 Blog
  - `2` 跳转 AI 工具
  - `3` 跳转 软件资源
  - `4` 跳转 影视网
- 主标题：修改 `landing-title` 文案。

## 6. 与后端联动建议

- 快捷入口中的内部链接（`/blog`、`/ai`、`/software`）可先用前端路由占位。
- 语录接口：`GET /api/v1/quote/random`
- 接口来源：后端代理 `https://api.mmp.cc/api/yiyan?format=text`，并做返回解析。

## 7. 注意事项

- `npm run dev` 与 `npm run build` 会自动执行背景清单同步脚本。
- 若 `public` 中没有 `bj*` 图片，会回退到 `/landing-bg.jpg`。
- 开发环境通过 Vite `proxy` 转发 `/api` 到 `http://localhost:8000`，需要后端服务已启动。

## 8. 端口说明（避免 ERR_UNSAFE_PORT）

- `6666` 在多数浏览器中属于不安全端口，访问 `http://localhost:6666` 会被拦截。
- 推荐开发端口：`5173` 或 `6677`。
- 若需固定使用 `6677`，请运行：`npm run dev:6677`。
